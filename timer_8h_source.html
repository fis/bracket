38 /**
39  * Low-level wrapper for handling a set of timers with a single `timerfd`.
40  *
41  * This class is intentionally low-level. Most clients should probably use this through a
42  * wrapper. The scheduling functions of event::Loop tie this to the event loop mechanisms. It's also
43  * possible to integrate this class in a GLib main loop.
44  *
45  * The semantics of this class are pretty simple: it holds a collection of \p PeriodicT objects
46  * attached to (unique) intervals, and a collection of \p OneshotT objects attached to delays. The
47  * Poll() method waits for at least one timer to expire (unless interrupted), then calls the
48  * provided function with all so far expired timers. You should probably arrange for Poll() to be
49  * called only when the file descriptor returned by #fd() indicates being ready to read.
50  *
51  * Periodic tags will be delivered at constant multiples of the rate, so using a rate of (e.g.) 1
52  * minute will cause a tag to be delivered right after the start of a new minute.
53  *
54  * \tparam PeriodicT object type for periodic timers
55  * \tparam OneshotT object type for one-shot timers
56  */
57 template <typename PeriodicT, typename OneshotT = PeriodicT>
