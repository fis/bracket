25 /**
26  * IRC connection.
27  *
28  * **Output flood control model**
29  *
30  * Every millisecond, we get one unit of credit, up to a credit limit of 10000 units (which is also
31  * the initial amount of credit after a connection has been opened). The cost of sending most
32  * messages is 1000 + N*10, where N is the number of bytes we need to send. An additional surcharge
33  * applies to some commands (nick, join, part, ping, userhost: 1000, topic, kick, mode: 2000, who:
34  * 3000). We can only send a message as long as we have sufficient credit for it.
35  *
36  * This model is implemented by keeping the following bits of state:
37  *
38  * - Amount of credit C at a particular time T.
39  * - A queue of bytes that need to be sent.
40  * - A queue of (length, per-message cost) pairs that describe the contents of the byte queue.
41  *
42  * When a client wants to send a message, do the following:
43  *
44  * - Serialize the message to the send queue, and compute its length and cost.
45  * - If the queue was previously empty, try to flush. (Otherwise we're already trying to clear the
46  * queue.)
47  *
48  * To flush the queue (when the socket seems ready for writing):
49  *
50  * - Update the credit counter (C += now-T, T = now).
51  * - See how many messages we can afford to send from the queue.
52  * - Try to write all those messages to the server.
53  * - Charge credit for the sent data. If the last message was incomplete, charge only the per-byte
54  * cost, but leave the per-command cost to be paid.
55  * - If the write stopped with EAGAIN/EWOULDBLOCK, start waiting for the descriptor to become ready
56  * for writing again.
57  * - Otherwise, if the queue still has data remaining, it must be because of insufficient
58  * credit. Compute a suitable delay until we can send the first message and start a timer. Once
59  * the timer fires, try flushing again.
60  *
61  * If we're not in the `kConnected` state, just let the messages remain in the queue. When a
62  * connection is established, try to flush the buffer. If connection is lost because of
63  * connect/read/write error, clear the queue.
64  */
