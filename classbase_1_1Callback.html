Detailed Description

Base class for callback interfaces.

Any callback interfaces registered to the callback containers (CallbackPtr, CallbackSet) must have this class as a base class. They should probably use virtual inheritance, in case the client wants to use a single class to implement more than one callback interface.

This class contains the support code for the callback to unregister itself from any callback container it's been added to when it gets destroyed. Conversely, callback containers know how to undo the linkage from their callbacks when they themselves get destroyed.
